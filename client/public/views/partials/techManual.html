Deployment

RASTA is implemented with the MEAN stack. It utilizes MongoDB for database operation, Express for routing, AngularJS for front end logic and Node JS for server operations. Below are the simple steps to deploy RASTA on a server.

  1. There are two required pieces of software need to be installed before the actual deplyment, which are Node JS and MongoDb.
    • MongoDb can be downloaded here. Once MongoDb is installed, the database server need to be started with the command mongod in the terminal.
    • Node can be downloaded here. Node comes with the NPM package manager that can be used to install additional packages.
  2. The best option to get the code for RASTA is through git, which comes pre-installed on many of the modern operating systems. The code is hosted on the JPL internal git server and can be accessed at https://code.jpl.nasa.gov/rhan/rasta_alt .
  3. Once the code is acquired onto the server, command npm install needs to be run under the root directory of the RASTA project to install all the necessary packages.
  4. At this point, the server can be started with the command node server. But for better background logging, an addition package called forever may be installed by running npm install forever. Once it is installed, the server can be started by running forever start server and information on the background process can be accessed by running forever list.
  5. The server can be stopped by killing the node process or running forever stop server.

Project Structure

RASTA Project

client

  • css: front end styling sheets
  • js: client side logic
  • views: front end pages

server

  • config: This folder contains server configuration data
  • database
    • managers: Code involving database operations
    • model: Mongoose schema models
  • logic: Server side logic including log parsing and service testing
  • routes: Routing code handling incoming calls

JSON Specifications

One of the ways to insert web service lists in to upload JSON configurations files in the following syntax:
  • The JSON file can contain one or more out of three top level attributes in the following format:
    { services: [ { service } ], functions: [ { function } ], async_services: [ { async_service } ] }
  • The above attributes contains lists of objects in the following formats
    • service: { name: String, url: String, response_type: String, type: String, //request type, e.g. 'get' or 'post' time_out: Number }
    • function: { name: String, services": [ { service } ] }
    • async_service: { name: String, url: String, response_type: String, type: String, //request type, e.g. 'get' or 'post', time_out: Number, checker_url: Sting //url to check job progress }
